## Summary

Цель: усиление DocOps-системы за счёт введения жёстких инвариантов и устранения дрейфа поведения LLM.

Цикл: **DocOps system hardening (determinism & constraints)**. Текущий заявленный этап lifecycle в рамках этой инициативы: **IDEA**. Основание: текущее состояние системы и выявленные системные отклонения (в т.ч. относительно boundary **DOA-FSN-002**).

Обязательный полный lifecycle (не сокращать):

`ANALYZE → IDEA → ARCH → ANALYZE → DEC → OP → IMP → VALIDATION`

---

## Context

Текущее состояние системы:

- lifecycle функционирует;
- snapshot boundary введён;
- validator и gate работают.

При этом выявлены системные отклонения:

- отсутствие **ANALYZE** перед **IDEA**;
- отсутствие второго **ANALYZE** перед **DEC**;
- отсутствие явного **canonical path mapping**;
- возможное изменение **prefix** (нарушение неизменяемости идентификаторов проекта);
- нарушение принципа **фактического IMP** (несоответствие снимка реализации коду/фактам);
- отсутствие достаточно строгих правил для среды **Cursor** / управляемых промптов.

---

## Problem

Система подвержена дрейфу из-за отсутствия жёстких контрактов:

- создание неканонических каталогов;
- генерация новых prefix вне установленной схемы;
- пропуск обязательных этапов lifecycle;
- несоответствие **IMP** фактической реализации.

Это приводит к:

- потере воспроизводимости;
- нарушению traceability;
- деградации доверия к системе.

---

## Goal

Сформировать набор обязательных правил, обеспечивающих:

- детерминированный lifecycle;
- стабильность структуры хранения документов;
- неизменяемость идентификаторов и префиксов в рамках политики DOA;
- соответствие документации реальному состоянию системы (включая фактичность IMP).

---

## Scope

### Included

- **ANALYZE** перед **IDEA**;
- **ANALYZE** перед **DEC**;
- **canonical path mapping** (явное сопоставление типов документов и каталогов);
- **prefix immutability** (запрет самовольной смены префикса / схемы ID);
- **IMP factuality** (IMP отражает фактическую реализацию и проверяемые артефакты);
- правила для **Cursor** (и прочих управляемых исполнителей), согласованные с DocOps.

### Excluded

- изменение бизнес-логики продукта вне DocOps;
- изменение существующих документов (в этом цикле — только create-only вперёд);
- рефакторинг кода вне рамок DocOps hardening.

---

## Definition of Done

Цикл считается завершённым, если:

1. зафиксированы **ARCH** и **DEC** для всех ключевых правил;
2. lifecycle включает два обязательных **ANALYZE** (до IDEA и до DEC);
3. устранено создание неканонических каталогов (механически и процессно);
4. исключена генерация новых prefix вне утверждённой политики;
5. **IMP** соответствует фактической реализации;
6. обеспечена воспроизводимость **prompt execution** (одинаковые входные условия → предсказуемый состав и размещение артефактов).

- снижение гибкости;
- необходимость строгого соблюдения правил всеми исполнителями (человек, агент, CI).
