## Title
Stabilize Document Creation Process (Filesystem + ID)

---

## Summary

В текущем состоянии системы DocOps Agent процесс создания документов
не является детерминированным.

Отсутствуют жёсткие инварианты для:
- структуры папок
- генерации ID
- именования файлов
- workflow создания документов

Это приводит к расползанию структуры и конфликтам.

---

## Problem

Процесс создания документов зависит от контекста и ручных действий.

Проявления:

- документы создаются в разных папках без строгого соответствия doc_type
- ID задаются вручную и могут конфликтовать
- naming используется непоследовательно
- workflow создания не формализован

---

## Goal

Стабилизировать процесс создания документов за счёт введения
базовых инвариантов системы.

---

## Scope

### Входит

- правила соответствия doc_type → folder
- правила генерации ID (через Cursor)
- правила именования файлов
- базовый workflow создания документа

### Не входит

- изменения validator или Document Gate
- автоматизация
- изменение существующих документов
- расширение DocOps модели

---

## Constraints

- create-only обязателен
- минимальная сложность
- без автоматизации на MVP этапе
- не изменять существующие документы

---

## Expected Outcome

- единая структура папок
- отсутствие конфликтов ID
- предсказуемый процесс создания документов
- улучшенная навигация по репозиторию

---

## Success Criteria

- пользователь не принимает решений о папке вручную
- ID не задаются вручную
- каждый документ создаётся по одному сценарию
- структура репозитория становится стабильной
