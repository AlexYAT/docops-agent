# 1. Goal

Опубликовать текущий baseline DocOps Agent MVP в GitHub и провести внешний оценочный аудит.

---

# 2. Scope

Включает:
- фиксацию snapshot baseline
- подготовку репозитория к публикации
- публикацию текущего состояния в GitHub
- проведение внешнего оценочного аудита
- фиксацию результатов аудита

Не включает:
- исправление всех non-critical замечаний до публикации
- автоматизацию процесса аудита
- новый этап реализации функциональности

---

# 3. Tasks

## DOA-OP-004-T01 — Зафиксировать baseline snapshot
**Description**
Сохранить implementation_snapshot текущего состояния после bootstrap-этапа как точку публикации.

**Done when**
- создан и сохранён implementation_snapshot
- snapshot ссылается на OP-001 и audit bootstrap-этапа

---

## DOA-OP-004-T02 — Подготовить GitHub publication
**Description**
Проверить репозиторий перед публикацией:
- README
- WORKFLOW
- структура docs/
- отсутствие случайных локальных файлов
- актуальность .gitignore

**Done when**
- репозиторий готов к публикации
- нет лишних временных файлов
- baseline готов к commit/push

---

## DOA-OP-004-T03 — Опубликовать baseline в GitHub
**Description**
Сделать commit/push текущего baseline в основной репозиторий.

**Done when**
- baseline опубликован в GitHub
- есть commit, на который можно ссылаться во внешнем аудите

---

## DOA-OP-004-T04 — Провести внешний оценочный аудит
**Description**
Передать опубликованный baseline на внешний review и собрать замечания:
- структура
- методология
- traceability
- качество шаблонов
- удобство практического использования

**Done when**
- внешний аудит проведён
- получены замечания/оценка

---

## DOA-OP-004-T05 — Зафиксировать результаты внешнего аудита
**Description**
Сохранить результаты внешней оценки в новом документе:
- audit_check и/или implementation_snapshot
- decision_log при необходимости

**Done when**
- результат внешнего аудита зафиксирован
- понятны next steps после аудита

# 5. Definition of Done

План считается выполненным если:
- baseline snapshot зафиксирован
- текущая версия опубликована в GitHub
- внешний аудит проведён
- результаты внешнего аудита документированы
- определены следующие действия на основе аудита

---

# 6. Notes
- внешний аудит оценочный, не блокирующий
- legacy issue не исправляется ретроспективно без отдельного решения
- публикация делается поверх уже пройденного bootstrap-аудита
