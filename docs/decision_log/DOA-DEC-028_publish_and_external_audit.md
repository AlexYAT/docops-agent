# Decision: DOA-DEC-028
Date: 2026-04-08
Status: accepted

## Title
Опубликовать текущий baseline в GitHub и провести внешний оценочный аудит

## Context
Bootstrap-этап DocOps Agent MVP достиг состояния, достаточного для внешнего просмотра:
созданы структура репозитория, шаблоны, sample documents, workflow и audit_check.
Критических нарушений не выявлено, что позволяет перейти к публикации текущего состояния и получению внешней оценки.

## Options considered
1. Продолжать внутреннюю доработку без публикации
2. Сначала опубликовать текущий baseline и получить внешний аудит
3. Сразу переходить к следующей реализации без внешней проверки

## Chosen solution
Сначала зафиксировать implementation snapshot, затем опубликовать baseline в GitHub и провести внешний оценочный аудит.

## Why
- позволяет получить внешнюю обратную связь на раннем этапе
- снижает риск продолжать развитие на скрыто ошибочной базе
- фиксирует проверяемый baseline проекта
- соответствует docs-first и traceability подходу

## Trade-offs
- потребуется дополнительный цикл оформления и публикации
- внешний аудит может вернуть замечания, которые замедлят следующий этап
- текущие non-critical и legacy моменты будут видны внешнему наблюдателю

## Follow-ups
- создать operational_plan для GitHub publication и external audit
- подготовить snapshot как baseline reference
- опубликовать текущее состояние репозитория
- собрать результаты внешнего аудита в отдельный audit_check / implementation_snapshot
