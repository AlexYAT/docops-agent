## Title

Полная русификация статических строк Report Generator без изменения deterministic-логики

## Context

В Report Generator уже добавлен человеко-читаемый deterministic слой (**`DOA-DEC-043`**, реализация по **`DOA-OP-024`**): заголовки разделов переведены на русский язык. Часть **статических** строк в теле отчёта по-прежнему на английском (пояснения к полям, фиксированные фразы в буллетах).

## Problem

Смешение русского и английского в одном отчёте снижает читаемость для русскоязычного читателя. При этом **нельзя** менять формат **source markers** `(source: …)`, детерминированное поведение, delta- и provenance-механику, а также ослаблять **forbidden**- и **DoD**-проверки (допускается лишь **адаптация** структурных проверок под русские строки, если это необходимо для совпадения с новым статическим текстом).

## Decision

Зафиксировать **полную русификацию** только **статических** строк отчёта:

- заголовков (уже частично на RU — довести политику до полного набора статических заголовков/подзаголовков отчёта, если появятся новые);
- поясняющих фраз;
- служебных текстов в буллетах (кроме перечисленных ниже исключений).

**Не изменять:**

- source markers `(source: …)` — формулировки и набор тегов;
- **ключи JSON** внутри backticks и сами сериализованные JSON-фрагменты как данные из входов;
- **delta**-логику (алгоритм сравнения и условия ветвлений);
- **provenance**-логику (источники полей, `git rev-parse`, fallback `unavailable`);
- **forbidden checks**;
- **DoD structural checks**, кроме точечной адаптации под русские статические строки при необходимости.

## Constraints

- Без NLP
- Без интерпретации содержимого findings
- Без рекомендаций и нормативных выводов в тексте отчёта
- Без изменения **anti-SoT** (отчёт не становится каноном)
- Только **deterministic text projection** (замена фиксированных шаблонных строк на русские эквиваленты)

## Rejected approaches

- Оставить **частичную** русификацию как целевое состояние
- Менять формат или семантику **source markers**
- Переводить **ключи JSON** или подменять структуру данных внутри отчёта
- Добавлять **аналитический** или интерпретационный слой

## Consequences

- Отчёт становится **единообразнее** и **понятнее** для русскоязычного чтения
- **Функционально** логика генератора не расширяется — меняется только слой представления статического текста
- Потребуется **точечный** smoke run и **новая** фиксация **`implementation_snapshot`** после реализации (create-only)

## Follow-ups

- Создать **`operational_plan`** на реализацию полной русификации статических строк
- Обновить **`scripts/doa_report_generator.py`**
- Выполнить smoke run
- Зафиксировать **`implementation_snapshot`**
