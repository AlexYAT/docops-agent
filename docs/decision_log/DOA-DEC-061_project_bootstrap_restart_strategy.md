## Title

Project Bootstrap Restart Strategy

---

## Context

В рамках запуска нового проекта (RAG MVP) был выполнен первый bootstrap-шаг: создан IDEA-документ с project-specific namespace (**RAG-IDEA-001**) и инициализирован git-репозиторий.

Цель проекта — демонстрационный, с высокими требованиями к чистоте DocOps-структуры.

На раннем этапе выявлены отклонения:

- использование project-specific namespace без зафиксированной стратегии;
- смешивание bootstrap-артефактов в первом коммите;
- отсутствие формализованного bootstrap flow для демонстрационного проекта.

---

## Problem

Текущий старт не соответствует требуемому уровню:

- чистоты структуры;
- демонстрационной прозрачности;
- воспроизводимости bootstrap-процесса.

Продолжение работы на этой базе приведёт к накоплению архитектурного долга.

---

## Decision

Принято решение: выполнить **полный restart** проекта перед продолжением DocOps-цикла.

---

## Scope of Restart

- удалить текущий проектный репозиторий;
- сохранить только **Pre-DocOps snapshot** как внешний input;
- не переносить существующие документы;
- не сохранять текущую git-историю.

---

## Bootstrap Strategy (новый старт)

Новый запуск должен:

1. использовать **system prompt snapshot** (**DOA-OTH-001**);
2. использовать **Pre-DocOps** как input (не как документ в каноне);
3. стартовать строго с **IDEA**;
4. использовать **согласованную namespace strategy** (будет определена отдельно);
5. соблюдать **atomic commits** с первого шага.

---

## Non-goals

- не выполнять частичную миграцию;
- не исправлять текущие документы в старом репозитории;
- не сохранять историю ошибок;
- не усложнять bootstrap дополнительными инструментами.

---

## Consequences

**Плюсы:**

- чистая демонстрационная история;
- корректная traceability с первого шага;
- отсутствие скрытых архитектурных конфликтов.

**Минусы:**

- потеря текущего локального состояния (приемлемо на этом этапе).

---

## Next Steps

1. Физически удалить текущий проект.
2. Подготовить чистый workspace.
3. Повторно запустить bootstrap через production prompt.
4. На этапе IDEA сразу контролировать namespace.
