# Decision: DOA-DEC-019

## Project
DocOps Agent (DOA)

## Date
2026-03-31

## Status
accepted

---

## Title
Выбрать workspace, к которому уже подключён MCP, как основной

---

## Context

Обнаружено наличие двух Notion workspace с одинаковыми названиями.

Cursor MCP работает с одним из них, что привело к:

- рассинхрону структуры
- созданию страниц "не там"
- ошибкам 404

---

## Problem

Поддержка двух workspace создаёт:

- путаницу
- нестабильный pipeline
- сложность отладки

---

## Chosen solution

Использовать workspace, к которому уже подключён Cursor MCP, как основной:

- очистить его
- создать новую структуру DocOps
- удалить/не использовать второй workspace

---

## Why

- минимальные изменения в текущей интеграции
- быстрее выйти на стабильный pipeline
- исключает ошибки авторизации

---

## Rule

В системе должен использоваться только один Notion workspace для DocOps
