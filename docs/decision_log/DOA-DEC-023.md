# Decision: DOA-DEC-023

## Project
DocOps Agent (DOA)

## Date
2026-03-31

## Status
accepted

---

## Title
Переход от разрозненной документации к управляемому DocOps pipeline

---

## Context

В ходе работы были созданы:

- 19+ decision_log
- несколько implementation_snapshot
- планы развития

Выявлено, что без агрегирования:

- сложно отслеживать состояние
- растёт когнитивная нагрузка
- появляется дублирование

---

## Problem

Отсутствие агрегированного состояния системы приводит к:

- потере контекста
- ошибкам в планировании
- усложнению работы с Cursor

---

## Chosen solution

Ввести правило:

- периодическая агрегация через implementation_snapshot
- использовать snapshot как “точку состояния системы”

---

## Rule

Актуальное состояние системы определяется последним implementation_snapshot, а не суммой всех документов
