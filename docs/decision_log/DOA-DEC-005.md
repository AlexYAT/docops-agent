# Decision: DOA-DEC-005

## Project
DocOps Agent (DOA)

## Date
2026-03-31

## Status
accepted (temporary)

---

## Title
Использовать Notion как временный основной storage документации (через Cursor + MCP)

---

## Context

Текущая система документации DOA требует:

- строгой структуры документов
- соблюдения planning layer
- удержания связей между документами

На практике это создаёт когнитивную нагрузку на этапе MVP.

Ранее рассматривался mindmap как способ навигации, но:

- требует отдельного инструмента
- не интегрируется с execution
- не решает проблему ввода данных

---

## Problem

Необходимо:

- упростить ввод документации
- не терять структуру полностью
- снизить нагрузку на пользователя

---

## Chosen Solution

Ввести временную архитектуру:

Cursor → MCP → Notion

Где:

- Cursor — точка создания (prompt → doc)
- MCP — execution layer
- Notion — основной storage документов

---

## Working Rules

1. Документы создаются через Cursor (не вручную в Notion)

2. MCP используется как слой доставки/сохранения

3. В Notion:
   - минимальное редактирование
   - только хранение и просмотр

4. Структура упрощённая:
   - без строгого соблюдения всех связей
   - без полной metadata схемы

5. Обязательная фиксация:
   - ключевые решения → decision_log
   - этапы → implementation_snapshot

---

## Why

- снимает когнитивную нагрузку
- ускоряет MVP
- позволяет работать в привычном workflow (Cursor-first)
- даёт единое хранилище

---

## Trade-offs

- временная потеря строгой структуры
- возможная деградация traceability
- зависимость от внешнего инструмента (Notion)

---

## Constraints

- не усложнять Notion (никаких сложных баз данных)
- не вводить автоматизацию на этом этапе
- не редактировать документы вручную (по возможности)

---

## Follow-ups

- вернуться к полноценной DocOps системе после MVP
- разработать миграцию из Notion → DocOps
- определить минимальную структуру страниц

---

## Notes

Это временный слой, а не финальная архитектура.

Цель — ускорение выхода на рабочую систему.
