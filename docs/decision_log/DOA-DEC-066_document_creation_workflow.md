## Title
Use fixed document creation workflow

---

## Context

После фиксации структуры папок, генерации ID и naming необходимо
зафиксировать единый workflow создания документов.

Без этого даже корректные правила могут применяться непоследовательно.

---

## Decision

Использовать фиксированный workflow создания документа.

---

## Workflow

Создание документа выполняется строго в следующем порядке:

1. Определить doc_type
2. Определить ID через сканирование репозитория
3. Сформировать имя файла (<ID>_<slug>.md)
4. Определить папку на основе doc_type
5. Создать файл в соответствующей папке
6. Добавить файл в git (git add)
7. Выполнить commit

---

## Why

- устраняет вариативность действий
- делает процесс воспроизводимым
- согласует все ранее принятые решения
- снижает риск ошибок

---

## Consequences

- пользователь не определяет порядок действий
- Cursor должен следовать фиксированной последовательности
- невозможно пропустить шаги
- каждый документ создаётся по единому сценарию

---

## Non-goals

- не автоматизировать workflow
- не добавлять дополнительные шаги
- не изменять существующие документы
