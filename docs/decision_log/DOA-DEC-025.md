## Title

Стандартизация артефактов MVP через Markdown-шаблоны

## Context

В архитектуре `DOA-ARCH-011` зафиксирован **markdown-only** репозиторий и перечень **doc_types**. Для bootstrap нужен единый механизм, чтобы новые документы имели предсказуемые поля и структуру без тяжёлой автоматизации.

## Options considered

- **Markdown templates** в `templates/<doc_type>/*.template.md` — копирование и заполнение вручную
- **Свободная форма** — только общий `README` и договорённости «по памяти»
- **YAML-only front matter + свободное тело** — структура метаданных да, но хуже читаемость для человека в диффах

## Chosen solution

Использовать **Markdown templates** как **основной** механизм стандартизации документов в MVP.

## Why

- Шаблоны **самодостаточны** для ручного заполнения и демонстрируют структуру doc_type
- Markdown остаётся **единым** форматом для тела и для процесса ревью в Git
- Минимальные затраты: не нужен парсер YAML-схем на этапе MVP

## Trade-offs

- Дисциплина остаётся **человеческой** (можно отклониться от шаблона без технического запрета)
- Дублирование блока Metadata в каждом файле — приемлемо для прозрачности

## Follow-ups
