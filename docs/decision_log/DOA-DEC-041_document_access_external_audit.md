## Title

Стандарт передачи документов во внешний аудит (доступ аудиторов к артефактам)

## Context

Внешний аудит архитектуры **не может** быть выполнен без доступа аудитора к **фактическому содержимому** проверяемых документов. Артефакты существуют **локально** в рабочей копии репозитория и **в удалённом** хранилище (например GitHub) после публикации коммитов.

## Problem

- Аудитор **не имеет** неявного доступа к файлам на машине инициатора аудита.
- При передаче только названий или общих отсылок без содержимого результат — **пустой** или недоказуемый аудит (как зафиксировано для `DOA-AUD-008` при отсутствии входных отчётов аудиторов).

## Options considered

1. **Ссылки на GitHub** (blob / raw по конкретному коммиту или ветке) — даёт доступ при условии публичности или прав доступа аудитора.
2. **Вставка полного текста** документов в запрос аудитора — гарантирует наличие содержимого в канале связи.
3. **Передача архива** с копиями файлов (или эквивалент прикреплённых файлов) — воспроизводимо при сохранении структуры путей или явном списке файлов.

## Chosen solution

Для каждого внешнего аудита инициатор **MUST** обеспечить доступ к содержимому **ровно одним или несколькими** способами из раздела «Allowed input formats». Использование только недоступных аудитору отсылок **MUST NOT** считаться допустимым входом.

## Allowed input formats

| Способ | Требование |
|--------|------------|
| **Ссылки на GitHub** | Предпочтительно: URL на конкретный файл в конкретном **коммите** (или иной однозначной ревизии), доступной аудитору. |
| **Вставка полного текста** | Полный текст каждого документа из Scope в теле запроса (или приложении), с явной меткой пути / `DOA-*`. |
| **Архив с файлами** | Архив (или набор вложений), содержащий копии файлов из Scope; перечень файлов **MUST** совпадать с Scope. |

Допускается **комбинация** способов при неизменном Scope.

## Constraints

- Аудитор **MUST** иметь возможность прочитать **полное** содержимое каждого документа из Scope (через ссылку, вставку или файл).
- **MUST NOT** ссылаться на ресурсы, к которым у аудитора **нет** доступа (приватные репозитории без учётных данных, временные URL, «посмотрите у себя»).
- **Scope** (перечень документов) **MUST** быть **явно** передан вместе с выбранным форматом доступа.

## Rejected approaches

- Формулировки вида **«документы есть локально»** без передачи содержимого или доступной ссылки — **отклонены**.
- Отсылка **«см. репозиторий»** без конкретных URL на ревизию и путь или без вставки текста — **отклонена**.

## Consequences

- Аудит становится **воспроизводимым**: третья сторона может проверить, на каком тексте основан вывод.
- **Исключаются** пустые проверки, вызванные отсутствием фактического входа.

## Follow-ups

- Провести **повторный** внешний аудит с передачей документов по настоящему стандарту и с последующей консолидацией в `audit_check` (create-only), согласованно с `DOA-DEC-040`.
