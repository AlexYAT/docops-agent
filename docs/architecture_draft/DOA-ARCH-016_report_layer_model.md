## Report Layer Model (DOA-ARCH-016)

## Summary

В системе DocOps Agent вводится слой отчетов (Report Layer), который не является частью Source of Truth, а формируется на основе документов.

## Concept

Отчет не является документом.

Отчет — это производный артефакт (derived artifact), который строится динамически на основе всех документов системы.

## Problem

- implementation_snapshot дублирует информацию
- отчеты устаревают
- create-only конфликтует с актуальностью

## Solution

Ввести разделение:

- Source of Truth → документы
- View Layer → отчет

Отчет:

- не хранится
- не версионируется
- всегда актуален

## Definition of Done

DoD считается выполненным, если:

- система проходит validator / gate
- на основе документов можно сгенерировать понятный человеку отчет

## Report Requirements

Отчет должен:

1. Объяснять что было сделано
2. Показывать связанные документы
3. Показывать статус системы
4. Быть понятным человеку без знания структуры DocOps

## Implications

- validator становится критическим компонентом
- структура документов должна быть строгой
- появляется основа для UI

## Notes

- отчет может быть реализован как CLI / Web UI
- не входит в MVP как обязательная реализация
- является архитектурным направлением
