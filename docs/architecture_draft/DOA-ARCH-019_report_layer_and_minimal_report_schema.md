## Title

Report Layer: Derived Artifacts Only and Minimal Report Schema (MVP)

## Purpose

Зафиксировать Report Layer как слой **только производных** артефактов, запретить отчёту роль документа или источника истины, задать минимальную схему отчёта MVP и обязательные поля происхождения (provenance). Документ **архитектурный**; UI, API и реализация не описываются.

## Scope

- Определение отчёта и границ Report Layer относительно канонических документов репозитория.
- Инварианты anti-SoT, запрещённые использования, условия недействительности отчёта.
- Минимальная схема отчёта и семантика обязательных разделов.
- Связь отчёта с выходами validator и gate (как входные данные для отображения, не как норма).

## Non-goals

- Форматы файлов, шаблоны, пайплайны генерации, хранение кэша.
- Web UI, CLI, библиотеки, схемы сериализации.
- Полная информационная модель отчёта после MVP.

## Report Layer definition

- **Отчёт** — производный артефакт (derived artifact), собранный из канонических документов и, при наличии, из **нормализованных** результатов validator / gate.
- Отчёт **MUST NOT** классифицироваться как `doc_type` репозитория DocOps и **MUST NOT** размещаться в `docs/<doc_type>/` как канонический документ.
- Отчёт **MUST** считаться воспроизводимым представлением состояния по заданному scope; при изменении источников или политики содержимое отчёта **MAY** измениться без версионирования как SoT.

## Anti-SoT invariants

- Отчёт **MUST NOT** быть Source of Truth для статусов, решений, планов или архитектуры.
- Отчёт **MUST NOT** порождать новые факты о системе: любое утверждение в отчёте **MUST** быть трассируемо к каноническому документу или к выводу validator/gate, зафиксированному вне отчёта (или к тому же scope документов).
- Канонические документы **MUST** оставаться единственным местом нормативной фиксации; отчёт **MUST** только агрегировать и пояснять.

## Forbidden usages

- Поля `Parent` и `Replaces` **MUST NOT** ссылаться на отчёт как на цель ссылки в смысле канона DocOps; отчёт **MUST NOT** выступать родителем или заменой для документа.
- Отчёт **MUST NOT** использоваться как основание для решений **без** ссылки на конкретные канонические документы (`DOA-*` в репозитории), подтверждающие утверждение.
- Отчёт **MUST NOT** подменять Policy Contract или расширять правила validator/gate.

## Required provenance fields

Каждый отчёт MVP **MUST** явно включать блок provenance со следующими полями:

| Поле | Смысл |
|------|--------|
| **generated timestamp** | Момент генерации отчёта (нормативная отметка времени). |
| **source scope** | Охват источников (например пути glob, список корней или явное описание множества файлов/типов). |
| **source commit or equivalent** | Идентификатор коммита репозитория или эквивалентная непротиворечивая ссылка на снимок источников. |
| **generator / version reference** | Идентификатор генератора и версии (достаточный для воспроизводимости интерпретации). |

Отсутствие любого из полей **MUST** приводить к классификации отчёта как недействительного (см. «Failure conditions»).

## Minimal report schema

MVP-отчёт **MUST** содержать ровно следующие верхнеуровневые разделы и не подменять их каноническими документами:

1. **summary** — краткое агрегированное состояние (производное).
2. **state by doc_type** — сводка по типам документов из scope (производное).
3. **risks** — риски и ограничения, выводимые из источников или из находок validator/gate (производное; без новых фактов вне трассировки).
4. **next steps** — предлагаемые шаги **только** как гипотеза для человека; **MUST NOT** трактоваться как решение без `decision_log` / канона.
5. **provenance** — блок обязательных полей из раздела «Required provenance fields».

Дополнительные разделы в MVP **MAY** отсутствовать.

## Minimum section semantics

- **summary** — не дублирует полный текст документов; **MUST** оставаться кратким обзором.
- **state by doc_type** — **MUST** отражать только состояние по данным scope; **MUST NOT** вводить типы или статусы, не подтверждённые каноническими файлами или политикой.
- **risks** — **MUST** быть помечены как производные; при использовании вывода validator/gate **MUST** сохраняться ссылка на соответствующий прогон или артефакт вне отчёта, если такой существует.
- **next steps** — **MUST NOT** иметь нормативной силы.
- **provenance** — **MUST** позволять читателю однозначно восстановить, на каком снимке источников построен отчёт.

## Relationship to canonical documents

- Канонические документы в `docs/<doc_type>/` **MUST** оставаться единственным источником норм и фактов о жизненном цикле и содержании артефактов.
- Отчёт **MUST** ссылаться на релевантные `DOA-*` идентификаторы там, где излагается смысл, зависящий от конкретного документа; утверждения без трассировки к канону **MUST NOT** включаться как факты.

## Relationship to validator and gate outputs

- Выводы validator и gate **MAY** входить в отчёт **только** как производные данные текущего или указанного в provenance прогона.
- Отчёт **MUST NOT** изменять семантику severity или правил блокировки; интерпретация политики **MUST** оставаться у Policy Contract и реализаций validator/gate.

## Failure conditions

Отчёт **MUST** считаться **недействительным**, если:

- отсутствует любой обязательный элемент provenance;
- не указан или неоднозначен **source scope**;
- не указан **source commit** (или эквивалент), при котором отчёт заявляет соответствие репозиторию;
- отчёт утверждает нормативные факты (статусы, решения, обязательность действий) без трассировки к каноническим документам;
- отчёт позиционируется как замена чтению канона (например единственный вход для принятия решения без ссылок на `DOA-*`).
