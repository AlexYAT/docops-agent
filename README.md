# DocOps Agent

## Что это и зачем

**DocOps Agent** — практика ведения разработки через **документы в репозитории**: идеи, архитектурные черновики, планы, решения, снимки состояния и аудиты лежат в каталоге `docs/` в согласованных форматах.

**Зачем:** уменьшить хаос и зависимость от «контекста в голове». Планы и решения **явные**, их можно найти, сравнить и проследить по цепочке связей.

## Ключевые концепции

| Концепция | Суть |
|-----------|------|
| **doc_types** | Типы артефактов (например `idea_to_mvp`, `architecture_draft`, `operational_plan`, `decision_log`, …). Каждый тип хранится в своей папке под `docs/` и имеет шаблон в `templates/`. |
| **create-only** | Содержательные изменения — через **новый файл и новый ID**, а не перепись уже зафиксированного документа. |
| **traceability** | У документов задаётся **родитель (Parent)** в метаданных; по цепочке видно, откуда взялось решение или план. |
| **planning layer** | Сначала стратегия и среднесрок (**longterm** / **midterm**), затем **operational_plan** с задачами; работа «вне плана» ограничена правилами bootstrap (см. [WORKFLOW.md](WORKFLOW.md)). |

## How to start

1. Ознакомьтесь с [WORKFLOW.md](WORKFLOW.md) — порядок этапов и правила.
2. Выберите нужный **doc_type** и скопируйте шаблон из `templates/<doc_type>/`.
3. Создайте новый файл в `docs/<doc_type>/` с уникальным именем вида `DOA-<TYPE>-<NNN>.md`.
4. Заполните блок **Metadata** (Project, Doc type, ID, Status, Date, Parent).
5. Ведите значимые решения отдельными записями в `docs/decision_log/`.
6. Перед закрытием этапа при необходимости оформите **audit_check** в `docs/audit_check/`.

## Пример цепочки

Упрощённо путь от идеи до проверки:

**idea** → **arch** → **plan** → **implementation** → **audit**

В терминах репозитория:

`idea_to_mvp` → `architecture_draft` → (`longterm_plan` опционально) → `midterm_plan` → `operational_plan` → `implementation_snapshot` → `audit_check`

Параллельно на любом шаге: **`decision_log`** для важных решений.

## Статус

**MVP** — bootstrap завершён: структура `docs/` и `templates/`, пользовательский `WORKFLOW.md`, примеры и аудиты в репозитории.
